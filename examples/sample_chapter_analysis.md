# 示例章节拆解：第一章 雨灯亮起

## 1. 一句话结构功能

本章用“欠条和木匣”建立主角目标，用雨灯街规则制造压迫，并在章尾用钥匙和禁令牵引下一章。

## 2. 开头状态

- 主角状态：带着父亲失踪后的未解问题进入陌生街区。
- 关键关系：主角与父亲之间存在未完成的信息传递。
- 主要矛盾：主角想找回木匣，但街区规则要求他先处理父亲留下的债。
- 读者期待：想知道雨灯街是什么、父亲押了什么、木匣里有什么。

## 3. 结尾状态

- 主角状态变化：从旁观者变成被规则追赶的人。
- 关系变化：父亲不再只是失踪者，而是留下债务和提示的人。
- 新目标/新危机：必须按钥匙提示寻找第三盏灯。
- 章尾钩子：“不要回头”把下一章行动变成即时风险。

## 4. 可复现模块卡

| 字段 | 内容 |
|---|---|
| 读者想看什么 | 主角如何在陌生规则中做出第一个选择 |
| 情绪链 | 好奇 -> 不安 -> 被迫选择 -> 小发现 -> 新禁令 |
| 戏剧单元 | 主角带目标进入异常空间，目标物变成规则入口 |
| 关键功能位 | 欠条 / 错位交易 / 追赶压力 / 禁令钩子 |
| 复现步骤 | 1. 给主角一个私人目标 2. 让环境用交易规则拦住他 3. 用目标物释放更大的问题 |
| 可替换项 | 街区可换成车站、档案馆、医院、旧学校 |
| 不可照搬 | 雨灯街、赎梦店、具体钥匙文案 |

## 5. 可学习规律

- 开篇奇观需要和主角私人目标绑定，避免只展示设定。
- 规则第一次出现时最好带有代价，让读者理解“不遵守会怎样”。
- 章尾钩子可以用禁令表达，比单纯抛谜题更容易推动下一章行动。

## 6. 不建议模仿

- 不要照搬雨灯街、欠条、赎梦等专属设定。
- 不要只堆奇观名词，必须让主角在规则中受到压力。
- 不要把父亲失踪的答案一次性解释完，应保留可推进的疑问。
