# 无 RAG / 仅 writing_guide / writing_guide + worldbuilding + memory 对比

本示例使用 `examples/demo_story.md` 和 `examples/sample_canonical_cards.json` 的安全素材，展示同一写作请求在不同知识条件下可能出现的差异。示例不是评测结论，也不承诺所有模型都会稳定提升。

## Same Request

```text
请生成一个原创“雨夜异常街区”题材的第一章提纲。
要求：有主角目标、规则压力、情绪推进和章尾钩子。
不要复制 demo 文本中的角色、店铺、专名或具体桥段。
```

## A. 不使用知识包

```markdown
主角来到一条神秘街道，发现这里有很多奇怪店铺。街道里的人都说话神秘，主角不断探索，最后发现街道隐藏着秘密。
```

观察：

- 章节状态变化：有“来到街道 -> 发现秘密”的粗略变化，但缺少明确行动目标。
- 冲突链：冲突偏概念化，缺少触发、升级和代价。
- 情绪链：有神秘感，但压力推进不清楚。
- 信息投放：容易一次性堆“奇怪店铺”“隐藏秘密”。
- AI 味：容易出现泛化词和空泛悬念。
- 设定一致性：没有明确世界规则，续写时容易漂移。
- 原作内容污染：污染风险较低，但也缺少可控约束。

## B. 使用 writing_guide

召回示例：

- `WR-001` 用私人目标承载奇观入口。
- `CP-001` 规则拦路型开篇冲突。
- `EM-001` 陌生规则压迫链。
- `AP-001` 避免只堆设定名词。

```markdown
主角为了取回一件与家人有关的物品进入雨夜街区。街区第一条规则阻止主角直接离开，要求主角完成一次带代价的交换。主角尝试绕开规则，但每一步都让目标更近、风险更清楚。结尾时，主角获得下一地点，同时收到一个禁令式提醒，为第二章留下压力。
```

观察：

- 章节状态变化：从“主动寻找”变为“被规则牵引”，更适合开篇。
- 冲突链：有目标、阻挡、代价、小回报和新钩子。
- 情绪链：能形成好奇、不安、被迫选择、小发现、新禁令。
- 信息投放：设定通过行动暴露，不需要长段说明。
- AI 味：泛化风险降低，但仍可能出现“神秘街区”等套话。
- 设定一致性：已有规则压力模板，但原创事实仍不充分。
- 原作内容污染：需要明确禁止复制 demo 专名、角色和桥段。

## C. 使用 writing_guide + worldbuilding + memory

额外上下文示例：

- `worldbuilding`：新作品的街区规则由用户原创确认，例如“每个路口只能用一次承诺换路标”。
- `memory`：上一段确认提纲已经建立“主角不能回应身后的声音”的章尾限制。

```markdown
开头：主角带着一个私人目标进入雨夜街区，目标与已确认的街区规则发生冲突。

中段：主角需要用一次承诺换取路标，但这个承诺会影响后续选择。规则不靠旁白解释，而是通过主角的失败尝试和旁人反应呈现。

转折：主角获得线索，却发现线索只在自己遵守禁令时有效。

结尾：主角抵达下一处地点，身后传来熟悉声音。由于 Memory 中已有“不能回应”的限制，结尾不立刻解除禁令，而是把选择压力留给下一章。
```

观察：

- 章节状态变化：不仅有开篇变化，也承接了上一轮确认信息。
- 冲突链：规则、代价和下一步行动更稳定。
- 情绪链：禁令与熟悉声音形成具体压力。
- 信息投放：世界观由已确认事实控制，减少临时编造。
- AI 味：仍取决于模型表达能力，但 RAG 能减少空泛组织。
- 设定一致性：worldbuilding 与 memory 能减少续写漂移。
- 原作内容污染：风险取决于知识卡质量；`writing_guide` 应只含抽象写法，原作事实不应导入为 `worldbuilding`。

## Practical Reading

这个对比展示的是“可解释约束”的价值：Agent 不只是多拿信息，而是能看到本次输出为什么用了某些结构、规则和连续性记忆。

限制也很明确：

- 当前检索是轻量 keyword / tag / card_type scoring，不是完整向量库。
- Canonical 合并依赖启发式和人工确认，可能误合并。
- 长文本最终质量仍受模型上下文、输出上限和用户确认质量影响。
